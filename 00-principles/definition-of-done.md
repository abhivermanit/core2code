# Definition of Done

A feature/task is considered **done** when all of the following are satisfied:

## Code Quality

- [ ] Code compiles without errors or warnings
- [ ] All linting rules pass
- [ ] Type checking passes
- [ ] No known security vulnerabilities introduced

## Testing

- [ ] Unit tests written and passing
- [ ] Integration tests written and passing (where applicable)
- [ ] Edge cases and error paths covered
- [ ] Test coverage meets project threshold

## Documentation

- [ ] Public APIs documented
- [ ] README updated (if applicable)
- [ ] ADR written (if architectural decision was made)
- [ ] Changelog updated

## Review

- [ ] Code reviewed by at least one team member
- [ ] Review feedback addressed
- [ ] No unresolved conversations

## Deployment

- [ ] CI/CD pipeline passes
- [ ] Feature works in staging environment
- [ ] No regressions in existing functionality
- [ ] Rollback plan documented (for significant changes)

## Acceptance

- [ ] Acceptance criteria from the user story are met
- [ ] Product owner has approved (where applicable)
